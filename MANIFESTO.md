# AURA Manifesto Draft

> Status: Draft. This is a proposed open source manifesto for review.

## The Interface Should Belong To The User

People should not have to reshape every thought into a text box. Speech, screenshots, camera frames, files, mobile captures, and browser context are all natural parts of thinking. AURA exists to make those signals usable.

- [ ] Make speech feel as immediate as typing.
- [ ] Make screenshots feel like context, not attachments.
- [ ] Make mobile devices useful extensions of the desktop.
- [ ] Make browser AI tools easier to compose.

## Local First Is A Design Choice

The local network is powerful. A phone and a desktop on the same desk should not need a remote server for every small act of cooperation.

- [ ] Prefer local discovery when possible.
- [ ] Prefer local processing when practical.
- [ ] Keep sensitive input close to the user.
- [ ] Make cloud services optional integrations, not mandatory foundations.

## AI Should Be A Layer, Not A Silo

AI products are useful, but they are often isolated. AURA should help people move context into and between them.

- [ ] Connect local inputs to AI pages.
- [ ] Support many AI surfaces instead of betting on one.
- [ ] Make adapters inspectable and community-maintained.
- [ ] Let users build workflows from small, understandable parts.

## Great Tools Start Small

AURA does not need to become grand all at once. It should earn trust by doing simple things reliably:

- [ ] Record voice.
- [ ] Transcribe locally.
- [ ] Send text to the active AI workflow.
- [ ] Send images from screen, camera, or phone.
- [ ] Explain what it is doing when something fails.

## The Long View

The dream is a personal AI mesh: devices, models, browsers, and tools cooperating around the user.

- [ ] AURA as a local input layer.
- [ ] AURA as a device bridge.
- [ ] AURA as a realtime context stream.
- [ ] AURA as an automation surface.
- [ ] AURA as an open local AI runtime.
